# 第一部分：总需求概览

## 1.1 项目背景

某汽车销售公司在抖音、快手等平台运营多个汽车销售直播间。直播间观众频繁提问关于车型的价格、配置、对比、推荐等问题。人工客服无法 7×24 小时响应，且知识范围受限于个人经验。

需要一个**智能问答平台**：
1. 自动识别直播间用户关于汽车的提问
2. 基于预设的车系知识库精准回答
3. 支持多轮对话，能理解上下文追问
4. 快速、准确、口播友好（回答格式适合主播念出来）

## 1.2 业务场景总览

```
┌─────────────────┐
│  直播间 / 客服   │  用户提问："秦PLUS落地多少钱？"
└────────┬────────┘
         │ 意图识别（车相关？）
         ▼
┌─────────────────┐
│  智能问答平台    │
│                  │
│  Skill 意图匹配  │──→ 问候/联系方式/活动 → 直接返回话术
│     │            │
│     ▼ 否         │
│  Tool 工具调用   │──→ 落地价/月供计算 → 调用计算器
│     │            │
│     ▼            │
│  RAG 检索增强    │──→ 向量化 → pgvector → LLM 生成
└────────┬────────┘
         │
         ▼
    "秦PLUS 55km领先型落地约9.5万，120km旗舰落地约16.4万…"
```

## 1.3 核心能力矩阵

| 能力 | 方式 | 生效条件 |
|------|------|---------|
| 打招呼 | GreetingSkill 正则匹配 | 输入含"你好/hi/在吗"等 |
| 联系方式 | ContactSkill 正则匹配 | 输入含"电话/微信/地址"等 |
| 活动查询 | ActivitySkill 正则匹配 | 输入含"优惠/贷款/置换"等 |
| 落地价计算 | CarPriceCalculator.onRoadPrice | 输入含"落地+车名" |
| 月供计算 | CarPriceCalculator.monthlyPayment | 输入含"月供/分期" |
| 车型咨询 | RAG 向量检索 + LLM | 其他车相关提问 |
| 多轮追问 | 历史会话注入 Prompt | 同一 sessionId 的连续提问 |

## 1.4 数据规模预估

| 维度 | 预估 |
|------|------|
| 品牌数 | 10+ |
| 车系数 | 50+ |
| 每车系文档大小 | 1.5~2 KB |
| 每车系 Chunk 数 | 5~8 个 |
| 知识库总量 | 2.5~4 万字符 |

## 1.5 技术栈速览

| 层 | 技术 |

# 第二部分：需求说明书

## 2.1 功能需求

### FR-001 车系文档管理

**需求描述**：运营人员可创建"卖车知识库"类型知识库，仅接受 `.md` 文件上传。

**验收标准**：
- [x] 创建知识库时可选择文档类型"卖车知识库（CAR_MD）"
- [x] 非 `.md` 文件上传时被拒绝，提示"此知识库仅支持 Markdown (.md) 文件"
- [x] 提供车系MD模板下载

### FR-002 Markdown 文档结构化分块

**需求描述**：MD 文件按 `##` 二级标题分块存入向量库，保持段落语义完整性。

**验收标准**：
- [x] frontmatter（`---` 包裹的 YAML）作为独立 chunk
- [x] 每个 `##` 章节独立成 chunk（≤500 字符）
- [x] 过长 `##` 章节按 `###` 拆分
- [x] 最终降级为通用分块策略

### FR-003 智能分层问答

**需求描述**：用户提问按优先级依次匹配：工具调用 → Skill 匹配 → RAG 检索。前两层不调 LLM，0 API 费用。

**验收标准**：

| 输入示例 | 预期路由 | 响应方式 |
|---------|---------|---------|
| `你好` | GreetingSkill | 预设话术（0 费用） |
| `你们电话多少` | ContactSkill | 预设话术（0 费用） |
| `现在有什么优惠` | ActivitySkill | 预设话术（0 费用） |
| `秦PLUS落地多少钱` | CarPriceCalculator | 工具计算+LLM润色 |
| `卡罗拉油耗多少` | RAG 检索 | Embedding检索+LLM生成 |

### FR-004 多轮会话

**需求描述**：同一 sessionId 下的问题共享上下文，AI 能理解代词指代和省略。

**验收标准**：
- [x] 首次提问自动创建会话
- [x] 后续提问传入 sessionId
- [x] 历史问答对以 UserMessage/AssistantMessage 注入 Prompt
- [x] 用户可查看、切换、新建会话

### FR-005 汽车销售专属话术

**需求描述**：卖车知识库的 Promt 自动切换为汽车销售顾问角色模板。

**验收标准**：
- [x] 检测 KB config 中 `docType:CAR_MD` → 自动切换 `CAR_SALES_SYSTEM_PROMPT`
- [x] 回答语气口语化、适合直播间口播
- [x] 支持车型咨询、价格查询、车型对比、预算推荐、适用场景分析

## 2.2 非功能需求

| 编号 | 类别 | 要求 |
|------|------|------|
| NFR-001 | 性能 | Skill 匹配 < 5ms，工具调用 < 50ms，RAG 完整流程 < 3s |
| NFR-002 | 可用性 | LLM/Embedding API 不可用时降级，不影响服务 |
| NFR-003 | 安全 | API Key 不入库（通过环境变量/配置文件注入并 excluded from git） |
| NFR-004 | 可扩展 | 新增车系只需上传 `.md` 文件；新增 Skill/Tool 只需添加 `@Component` 类 |
| NFR-005 | 数据 | 问答记录持久化（question + answer 双表），支持历史回溯 |

## 2.3 用户角色

| 角色 | 权限 |
|------|------|
| ADMIN | 全部权限：用户管理、知识库管理、文档管理、问答 |
| KNOWLEDGE_BASE_ADMIN | 知识库管理、文档管理、问答 |
| 普通用户 | 仅问答 |

```

          │             │             │
```

## 3.3 MD 文档分块策略

```
MD 文件
  │
  ├─ --- YAML frontmatter --- → 独立 chunk #1
  │   (brand/model/price/tags...)
  │
  └─ 正文
      │
      ├─ ## 基本信息           → chunk #2 (≤500字符)
      ├─ ## 配置与价格         → chunk #3 (含Markdown表格)
      ├─ ## 动力与油耗         → chunk #4
      ├─ ## 核心卖点           → chunk #5
      ├─ ## 适合人群           → chunk #6
      └─ ## 竞品对比           → chunk #7
```

**关键规则**：
1. frontmatter 作为独立 chunk，便于"秦PLUS多少钱"这类元数据查询
2. 价格表格完整保留，不拆散
3. 过长章节（>500字符）按 `###` 子标题拆分
4. 最终降级为通用 `smartChunkContent()` 策略

## 4.2 技术实现要点

### 4.2.1 智能意图分层

**执行顺序**：工具调用 > Skill 匹配 > RAG 检索

```java
// AgentService.answer() 核心逻辑
public AnswerResponse answer(String question, UUID knowledgeBaseId, String sessionId) {
    // ① 工具调用（最高优先级）
    String toolResult = callToolsIfNeeded(question);
    if (toolResult != null) {
        return callLLMWithContext(toolResult, question, knowledgeBaseId, sessionId);
    }

    // ② Skill 匹配（正则，0 API 费用）
    AnswerResponse skillResp = skillClassifier.matchAndExecute(question);
    if (skillResp != null) return skillResp;

    // ③ 完整 RAG 流程
    return callRAG(question, knowledgeBaseId, sessionId);
}
```

### 4.2.2 车价计算工具

**工具识别正则**：
- 落地价：`.*(落地多少钱|落地价|上路多少钱|全办下来).*(车名关键词)`
- 月供：`.*(月供|分期|按揭|首付).*(车名关键词)`

**定价库**：静态 `HashMap<车名, double[] 配置价格列表>`，覆盖 39 款车

**计算逻辑**：
```
落地价 = 指导价 + 购置税(指导价/11.3) + 保险(4000+指导价×1%) + 上牌(500)
月供   = 等额本息公式：贷款额×月利率×(1+月利率)^N / ((1+月利率)^N - 1)
```

### 4.2.3 多轮对话上下文注入

```java
// Prompt 构建时注入历史
List<Message> messages = new ArrayList<>();
messages.add(new SystemMessage(SYSTEM_PROMPT));

if (sessionId != null) {
    List<Question> history = questionMapper.findBySessionIdOrderByCreatedAt(sessionId);
    for (Question q : history) {
        Answer a = answerMapper.findByQuestionId(q.getId()).orElse(null);
        messages.add(new UserMessage(q.getQuestion()));      // ← 历史问题
        if (a != null) messages.add(new AssistantMessage(a.getAnswer())); // ← 历史答案
    }
}
messages.add(new UserMessage("检索上下文 + 当前问题"));
```

### 4.2.4 降级策略矩阵

| 故障场景 | 降级行为 |
|---------|---------|
| DashScope Embedding API 不可用 | hash 伪随机向量（相同文本输出相同向量） |
| DeepSeek Chat API 不可用 | 返回检索到的原始文本片段 |
| pgvector 无匹配结果 | 提示"未找到相关内容" |
| JSON 序列化失败 | sources 写入空数组 `[]`，不阻断业务流程 |

|-----------|--------|---------|
| `{"docType":"CAR_MD"}` | `CAR_SALES_SYSTEM_PROMPT` | 汽车销售顾问，口播友好 |

## 4.3 新增 Skill/Tool 开发指引

### 新增一个 Skill（以"试驾预约Skill"为例）

```java
@Component
public class TestDriveSkill implements AgentSkill {

    private static final List<Pattern> PATTERNS = List.of(
        Pattern.compile(".*(我想试驾|预约试驾|试驾.*预约|能不能试驾).*")
    );

    @Override
    public boolean canHandle(String input) {
        return PATTERNS.stream().anyMatch(p -> p.matcher(input.trim()).matches());
    }

    @Override
    public AnswerResponse execute(String input) {
        return AnswerResponse.builder()
            .answer("🚗 试驾预约已为您登记！稍后将有销售顾问联系您确认时间～")
            .sources(List.of())
            .confidence(1.0f)
            .build();
    }
}
```

Spring 自动发现 → `SkillClassifier` 自动注入 → 无需改任何其他代码。

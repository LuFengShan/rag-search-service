# RAG知识库智能问答系统 - 一键部署脚本使用说明

## 📋 脚本功能

`deploy.sh` 是一个一键部署脚本，实现以下功能：

1. ✅ **环境检测** - 自动检查 JDK、Maven、PostgreSQL、Redis 版本
2. ✅ **依赖检查** - 确保所有依赖满足最低版本要求
3. ✅ **数据库初始化** - 自动创建数据库和启用 pgvector 扩展
4. ✅ **Maven 打包** - 自动构建项目
5. ✅ **应用启动** - 启动 RAG 应用并等待健康检查
6. ✅ **进程管理** - 支持启动、停止、重启、查看状态

## 🚀 快速开始

### 一键部署（完整流程）

```bash
cd deployment
./deploy.sh
```

脚本将自动执行：
1. 环境检查（JDK 21+、Maven 3.6+、PostgreSQL 15+、Redis 6+）
2. Maven 构建
3. 数据库初始化
4. 应用启动

### 查看帮助

```bash
./deploy.sh --help
```

## 📖 使用示例

### 1. 首次部署

```bash
# 完整部署
./deploy.sh
```

### 2. 仅检查环境

```bash
# 检查环境但不构建和启动
./deploy.sh --skip-build --skip-start
```

### 3. 跳过环境检查

```bash
# 跳过环境检查，直接构建
./deploy.sh --skip-checks
```

### 4. 重启应用

```bash
# 停止旧进程，重新构建并启动
./deploy.sh --restart
```

### 5. 停止应用

```bash
# 停止 RAG 应用
./deploy.sh --stop
```

### 6. 查看状态

```bash
# 查看应用运行状态
./deploy.sh --status
```

### 7. 查看日志

```bash
# 实时查看应用日志
tail -f logs/rag-app.log

# 查看最近 100 行日志
tail -100 logs/rag-app.log

# 搜索错误日志
grep ERROR logs/rag-app.log
```

## 🔍 环境要求

脚本会自动检测以下环境：

| 组件 | 最低版本 | 说明 | 必需 |
|------|---------|------|------|
| JDK | 21 | Java运行环境 | ✅ |
| Maven | 3.6 | 项目构建工具 | ✅ |
| PostgreSQL | 15 | 关系型数据库 | ✅ |
| pgvector | 0.5 | 向量数据库扩展 | ✅ |
| Redis | 6 | 缓存服务 | ❌ |

如果检测到版本过低，脚本会报错并拒绝继续执行。

## 🎯 版本检测逻辑

脚本会：
1. 检查命令是否存在
2. 获取版本号
3. 与最低版本要求比较
4. 版本不满足时显示错误信息和建议

示例输出：

```
[INFO] 检查 Java 环境...
[SUCCESS] Java 版本: 21.0.3 ✓

[INFO] 检查 Maven 环境...
[SUCCESS] Maven 版本: 3.9.6 ✓

[INFO] 检查 PostgreSQL 环境...
[SUCCESS] PostgreSQL 版本: 15.5 ✓

[INFO] 检查 pgvector 扩展...
[SUCCESS] pgvector 扩展已安装 ✓

[INFO] 检查 Redis 环境...
[WARN] 未检测到 Redis 安装 (可选组件)
```

## ⚙️ 配置说明

### 数据库配置

脚本默认配置：
- 数据库名: `example_db`
- 用户名: `postgres`
- 密码: `123456`
- 主机: `localhost`
- 端口: `5432`

如需修改，编辑 `backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/example_db
    username: postgres
    password: your-password
```

### JVM 参数

脚本默认 JVM 参数：
- 初始堆内存: 512MB (`-Xms512m`)
- 最大堆内存: 1024MB (`-Xmx1024m`)
- GC算法: G1GC (`-XX:+UseG1GC`)

如需修改，编辑 `deploy.sh` 中的 `java` 命令：

```bash
java -Xms1g -Xmx2g -XX:+UseG1GC \
    -jar "$jar_file" \
    >> "$SCRIPT_DIR/logs/rag-app.log" 2>&1 &
```

## 🔧 故障排查

### 1. 端口被占用

```bash
# 查找占用端口的进程
lsof -i :8080

# 停止占用进程
kill -9 <PID>

# 或使用其他端口启动应用
```

### 2. 数据库连接失败

```bash
# macOS 启动 PostgreSQL
brew services start postgresql@15

# Ubuntu 启动 PostgreSQL
sudo systemctl start postgresql

# CentOS 启动 PostgreSQL
sudo systemctl start postgresql
```

### 3. 应用启动失败

```bash
# 查看错误日志
tail -100 logs/rag-app.log

# 检查 JAR 文件是否存在
ls -lh backend/target/rag-1.0.0.jar
```

### 4. Maven 构建失败

```bash
# 清理并重新构建
cd backend
mvn clean
cd ..
./deploy.sh
```

## 🔐 安全建议

### 1. 修改默认密码

首次部署后，请立即修改以下密码：

```bash
# 修改 PostgreSQL 密码
psql -U postgres -c "ALTER USER postgres WITH PASSWORD 'your-strong-password';"

# 修改应用配置文件
vim backend/src/main/resources/application.yml
```

### 2. 使用环境变量

生产环境建议使用环境变量：

```bash
export SPRING_DATASOURCE_PASSWORD=your-password
export JWT_SECRET=your-64-char-secret-key
./deploy.sh
```

### 3. 启用 HTTPS

生产环境请配置 HTTPS，可参考 `单机部署指南.md`。

## 📝 常见问题

### Q1: 脚本执行时报错 "Permission denied"

**解决**：
```bash
chmod +x deploy.sh
```

### Q2: 提示 "command not found"

**原因**：相关命令未安装或未在 PATH 中

**解决**：参考 `单机部署指南.md` 安装相应软件

### Q3: pgvector 扩展安装失败

**macOS**：
```bash
brew install pgvector
```

**Ubuntu**：
```bash
sudo apt install postgresql-15-pgvector
```

**CentOS**：
```bash
sudo dnf install pgvector_15
```

### Q4: 数据库连接被拒绝

**解决**：
1. 确保 PostgreSQL 服务已启动
2. 检查 pg_hba.conf 配置
3. 验证用户名密码正确

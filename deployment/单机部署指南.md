# RAG知识库智能问答系统 - 单机部署指南

## 一、环境要求

### 1.1 硬件要求
- **CPU**: 2核以上（推荐4核）
- **内存**: 4GB以上（推荐8GB）
- **硬盘**: 20GB以上可用空间
- **操作系统**: macOS、Linux (Ubuntu/CentOS)

### 1.2 软件依赖

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| Java | 21+ | 后端服务运行环境 |
| Maven | 3.6+ | 项目构建工具 |
| PostgreSQL | 15+ | 关系型数据库 |
| PostgreSQL + pgvector | 最新版 | 向量数据库扩展 |
| Redis | 6+ | 缓存和会话存储（可选） |

## 二、环境安装

### 2.1 安装Java 21

**macOS:**
```bash
# 使用Homebrew安装
brew install openjdk@21

# 配置环境变量
echo 'export JAVA_HOME=$(/usr/libexec/java_home -v 21)' >> ~/.zshrc
echo 'export PATH="$JAVA_HOME/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

**Linux (Ubuntu):**
```bash
# 添加 Adoptium 仓库
wget -qO - https://packages.adoptium.net/artifactory/api/gpg/key/public | sudo tee /etc/apt/keyrings/adoptium.asc
echo "deb [signed-by=/etc/apt/keyrings/adoptium.asc] https://packages.adoptium.net/artifactory/deb $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/adoptium.list

# 安装 Java 21
sudo apt update
sudo apt install temurin-21-jdk

# 验证安装
java -version
```

**Linux (CentOS/RHEL):**
```bash
# 添加 EPEL 和 Adoptium 仓库
sudo dnf install -y https://packages.adoptium.net/artifactory/rpm/epel/$(uname -m)/adoptium.repo

# 安装 Java 21
sudo dnf install -y temurin-21-jdk

# 验证安装
java -version
```

### 2.2 安装Maven

**macOS:**
```bash
brew install maven
mvn -version
```

**Linux:**
```bash
# Ubuntu
sudo apt install maven

# CentOS
sudo dnf install maven

# 验证安装
mvn -version
```

### 2.3 安装PostgreSQL 15+ with pgvector

**macOS:**
```bash
# 使用Homebrew安装
brew install postgresql@15
brew install pgvector

# 启动PostgreSQL服务
brew services start postgresql@15

# 连接数据库
psql -U $(whoami) -d postgres
```

**Linux (Ubuntu):**
```bash
# 安装 PostgreSQL
sudo apt install postgresql postgresql-contrib

# 安装 pgvector
sudo apt install postgresql-15-pgvector

# 启动服务
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 切换到postgres用户
sudo -u postgres psql
```

**Linux (CentOS):**
```bash
# 安装 PostgreSQL 和 pgvector
sudo dnf install -y postgresql-server postgresql-contrib
sudo dnf install -y pgvector_15  # 根据实际版本调整

# 初始化数据库
sudo postgresql-setup --initdb

# 启动服务
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 连接数据库
sudo -u postgres psql
```

## 三、数据库配置

### 3.1 创建数据库和用户

```sql
-- 连接到PostgreSQL
psql -U postgres

-- 创建数据库
CREATE DATABASE example_db;

-- 创建用户（可选，使用默认postgres用户也行）
CREATE USER raguser WITH PASSWORD 'ragpassword';
GRANT ALL PRIVILEGES ON DATABASE example_db TO raguser;

-- 切换到数据库
\c example_db

-- 启用pgvector扩展
CREATE EXTENSION IF NOT EXISTS vector;

-- 授予用户权限
GRANT ALL ON SCHEMA public TO raguser;
```

### 3.2 验证pgvector扩展

```sql
-- 验证扩展是否安装成功
SELECT * FROM pg_extension WHERE extname = 'vector';

-- 测试创建向量表
CREATE TABLE test_vector (
    id SERIAL,
    embedding vector(1536)
);

-- 测试插入和查询
INSERT INTO test_vector (embedding) VALUES ('[0.1,0.2,0.3]');
SELECT * FROM test_vector ORDER BY embedding <=> '[0.1,0.2,0.3]' LIMIT 1;

-- 删除测试表
DROP TABLE test_vector;
```

## 四、项目配置

### 4.1 修改数据库连接配置

编辑 `backend/src/main/resources/application.yml`:

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/example_db
    username: postgres          # 或 raguser
    password: 123456            # 或 ragpassword
    driver-class-name: org.postgresql.Driver
  
  jpa:
    hibernate:
      ddl-auto: update          # 首次部署使用update，生产环境建议manual
    show-sql: false
    properties:
      hibernate:
        dialect: org.hibernate.dialect.PostgreSQLDialect

  data:
    redis:
      host: localhost
      port: 6379
      password:                 # 如果Redis设置了密码，填写在这里
```

### 4.2 修改JWT密钥（生产环境必须）

```yaml
jwt:
  secret: your-256-bit-secret-key-here-must-be-at-least-32-characters
  # 建议使用随机生成的64位密钥
  # 可以使用以下命令生成：
  # openssl rand -base64 64
  expiration: 86400000          # 24小时过期时间
```

### 4.3 日志配置

日志文件会输出到 `./logs/` 目录：

```yaml
logging:
  level:
    com.example.rag: DEBUG
    com.example.rag.controller: INFO
    com.example.rag.service: INFO
    org.springframework.security: INFO
  file:
    name: ./logs/rag-app.log
```

## 五、构建和部署

### 5.1 构建项目

```bash
# 进入后端目录
cd backend

# 清理并构建
mvn clean package -DskipTests

# 或仅编译
mvn clean compile
```

构建成功后，会在 `target/` 目录下生成 `rag-1.0.0.jar` 文件。

### 5.2 运行应用

**开发环境运行：**
```bash
mvn spring-boot:run
```

**生产环境运行：**
```bash
# 后台运行
nohup java -jar target/rag-1.0.0.jar --server.port=8080 > logs/app.log 2>&1 &

# 或使用nohup重定向输出
nohup java -Xms512m -Xmx1024m -jar target/rag-1.0.0.jar > logs/app.log 2>&1 &

# 查看启动日志
tail -f logs/app.log

# 检查进程
ps aux | grep rag-1.0.0.jar
```

### 5.3 验证部署

```bash
# 检查健康状态
curl http://localhost:8080/actuator/health

# 访问API文档
# 浏览器打开: http://localhost:8080/swagger-ui.html
# 或: http://localhost:8080/api-docs
```

## 六、系统调优

### 6.1 JVM参数调优

编辑启动脚本或直接使用命令行参数：

```bash
# 最小堆内存512MB，最大1GB
java -Xms512m -Xmx1024m \
     -XX:+UseG1GC \
     -XX:MaxGCPauseMillis=200 \
     -jar target/rag-1.0.0.jar
```

**生产环境推荐配置：**
```bash
java -Xms2g -Xmx2g \
     -XX:+UseG1GC \
     -XX:MaxGCPauseMillis=200 \
     -XX:+HeapDumpOnOutOfMemoryError \
     -XX:HeapDumpPath=./logs/heapdump.hprof \
     -jar target/rag-1.0.0.jar
```

### 6.2 数据库连接池配置

在 `application.yml` 中添加：

```yaml
spring:
  datasource:
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      idle-timeout: 300000
      connection-timeout: 20000
      max-lifetime: 1200000
```

### 6.3 日志文件管理

配置日志滚动，避免单个文件过大：

```yaml
logging:
  logback:
    rollingpolicy:
      max-file-size: 10MB
      max-history: 30
      total-size-cap: 1GB
```

## 七、常见问题排查

### 7.1 数据库连接失败

**问题：** `Connection refused` 或 `Authentication failed`

**解决方案：**
1. 检查PostgreSQL服务是否启动：
   ```bash
   # macOS
   brew services list | grep postgresql
   
   # Linux
   systemctl status postgresql
   ```

2. 检查pg_hba.conf配置，允许本地连接：
   ```
   # 添加以下行
   host    all             all             127.0.0.1/32            md5
   ```

3. 重启PostgreSQL服务：
   ```bash
   sudo systemctl restart postgresql
   ```

### 7.2 向量查询失败

**问题：** `relation "document_chunk" does not exist`

**解决方案：**
1. 确认pgvector扩展已启用：
   ```sql
   CREATE EXTENSION IF NOT EXISTS vector;
   ```

2. 重启应用，Hibernate会自动创建表

### 7.3 内存不足

**问题：** `OutOfMemoryError`

**解决方案：**
1. 增加JVM堆内存大小
2. 检查服务器可用内存
3. 优化数据库查询，减少数据量

### 7.4 端口被占用

**问题：** `Port 8080 was already in use`

**解决方案：**
```bash
# 查找占用端口的进程
lsof -i :8080

# 停止占用进程
kill -9 <PID>

# 或使用其他端口启动
java -jar target/rag-1.0.0.jar --server.port=8081
```

## 八、备份与恢复

### 8.1 数据库备份

```bash
# 备份数据库
pg_dump -U postgres -d example_db -F c -b -v -f backup_$(date +%Y%m%d).dump

# 备份到远程服务器
pg_dump -U postgres -d example_db | gzip | ssh user@backup-server 'cat > backup_$(date +%Y%m%d).sql.gz'
```

### 8.2 数据库恢复

```bash
# 恢复数据库
pg_restore -U postgres -d example_db -c backup_20240517.dump

# 或先删除再恢复
dropdb -U postgres example_db
createdb -U postgres example_db
pg_restore -U postgres -d example_db backup_20240517.dump
```

## 九、安全建议

### 9.1 生产环境必做

1. **修改默认密码**：立即更改所有默认用户的密码
2. **JWT密钥**：使用足够长度的随机密钥
3. **数据库密码**：使用强密码并定期更换
4. **HTTPS**：配置SSL/TLS证书
5. **防火墙**：只开放必要端口（80, 443, 8080）
6. **定期更新**：及时更新Java和依赖库的安全补丁

### 9.2 可选安全措施

```yaml
# 启用HTTPS
server:
  port: 8443
  ssl:
    key-store: classpath:keystore.p12
    key-store-password: your-password
    keyStoreType: PKCS12
    keyAlias: your-alias
```

## 十、监控与日志

### 10.1 监控端点

系统已集成Spring Boot Actuator：

- 健康检查：`GET /actuator/health`
- 信息查看：`GET /actuator/info`
- 指标查看：`GET /actuator/metrics`
- 环境变量：`GET /actuator/env`

### 10.2 日志文件位置

```
backend/
├── logs/
│   ├── rag-app.log      # 主日志
│   ├── errors.log       # 错误日志
│   ├── requests.log     # 请求日志
│   └── services.log     # 服务日志
```

### 10.3 日志分析示例

```bash
# 查看错误日志
tail -100 logs/errors.log | grep ERROR

# 查看特定时间的日志
grep "2024-05-17 16:" logs/rag-app.log

# 查看慢请求
grep "Duration" logs/requests.log | awk '{if($NF>1000) print}'
```
